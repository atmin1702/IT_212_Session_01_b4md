# BÀI 4: Phan tích & Lựa chọn (Tối ưu hóa hiệu năng)

### 1. Đáp án lựa chọn
**Prompt 2:** "Hãy chuyển đổi hàm Java duyệt mảng bằng vòng lặp for dưới đây sang sử dụng Java Stream API để code ngắn gọn và dễ bảo trì hơn. Giải thích lý do tối ưu."

---

### 2. Lý do chọn dựa trên tiêu chuẩn thiết kế prompt

**Prompt 2** mang lại kết quả tối ưu và chính xác nhất vì nó đáp ứng đầy đủ các tiêu chuẩn thiết kế prompt nâng cao (Lesson 2):
* **Mục tiêu rõ ràng (Goal):** Chỉ định cụ thể hành động cần làm là chuyển đổi từ vòng lặp `for` truyền thống sang **Java Stream API**.
* **Ràng buộc chặt chẽ (Constraints):** Định hướng tiêu chí đầu ra rõ ràng là mã nguồn phải "ngắn gọn và dễ bảo trì hơn", đồng thời yêu cầu AI phải giải thích lý do tối ưu thay vì chỉ quăng ra một đoạn code mẫu.

**Lý do Prompt 1 chưa tốt:**
* **Mơ hồ và thiếu ngữ cảnh:** Câu lệnh "tối ưu hóa... giúp tôi chạy nhanh hơn" quá chung chung. AI sẽ không biết người dùng muốn tối ưu theo hướng nào (giảm độ phức tạp thuật toán, viết ngắn gọn lại, hay tối ưu bộ nhớ).
* **Sai lệch mục tiêu nghiệp vụ:** Với một danh sách số nguyên đơn giản, việc đổi cách viết không làm thay đổi đáng kể tốc độ chạy (hiệu năng phần cứng), mà mục đích thực tế ở đây là tối ưu cấu trúc code (clean code). Prompt 1 dễ khiến AI phản hồi lan man hoặc đưa ra các giải pháp phức tạp hóa vấn đề không cần thiết.
